## üîÑ Documentation Maintenance

### When to Update

| Trigger            | Files to Update                              |
| ------------------ | -------------------------------------------- |
| New feature        | Feature doc + README + CHANGELOG             |
| Bug fix            | CHANGELOG                                    |
| Code refactor      | ARCHITECTURE (if structure changes)          |
| Version release    | CHANGELOG + README (version badge)           |
| API changes        | README (API section) + affected feature docs |
| Contribution rules | CONTRIBUTING.md                              |
| Breaking changes   | CHANGELOG + Migration guide                  |

### Documentation Checklist

When adding a new feature:

- [ ] Create feature documentation file
- [ ] Update README.md
- [ ] Update CHANGELOG.md
- [ ] Update COMPLETE_IMPLEMENTATION_SUMMARY.md
- [ ] Update this index (DOCUMENTATION_INDEX.md)
- [ ] Add to QUICKSTART.md if relevant
- [ ] Update ARCHITECTURE.md if structure changes
